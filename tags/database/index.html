Tag: Database

  • What the heck is a Database Index? — When you do a standard select * from table_name where columns=some_value from a table.. the database has no idea where IN the table that data exists. In order to figure out which rows in the table it needs to return to you (based on your where clause), it looks through the rows in the table. Of course, if you don’t set a limit clause, it has to look through EVERY single entry in the table because it doesn’t know where/how-many instances of a particular value it might find.
